# CryptRepublic Wave 2 — Authentication + Accounts — Implementation Plan

> For agentic workers: REQUIRED SUB-SKILL — before touching code, load `superpowers:test-driven-development` and `superpowers:executing-plans`. Every task below is TDD: write the failing test, run it and SEE it fail, implement the real code, run it and SEE it pass, then commit. No step may be skipped or reordered. Never write implementation before its test. Never invent APIs — match the exact signatures in each task's **Interfaces** block.

**Goal.** Deliver dual authentication for CryptRepublic — email + passphrase (Argon2id hashing, opaque DB-backed httpOnly sessions) and SIWE (EIP-4361, server-verified single-use nonce) — over one non-custodial identity model, plus the citizenship-application seed (DRAFT application on register). Port `Auth.html` faithfully to `app/auth/page.tsx` wired to the real API, add a minimal session-guarded `/dashboard`, and route register → `/dashboard/mint`. All of this satisfies the Wave 2 row of spec §9 and the auth rules of spec §4.

**Architecture.** Next.js 15 App Router route handlers under `app/api/auth/*` and `app/api/applications/*` are the only network surface. They are server-only, Zod-validate every input (reject unknown fields), enforce an Origin/Referer allowlist on every POST, and are guarded by `requireSession()` where noted. Credentials never leave the server as anything but an Argon2id hash (`User.passwordHash`) and short-TTL SIWE nonces (`SiweNonce`). Sessions are opaque: a 256-bit random token lives in an httpOnly `cr_session` cookie; only its SHA-256 (`Session.tokenHash`, unique) is stored. Route handlers read the cookie via `getSessionFromRequest(req)` (parses `Cookie` header) and write/clear it via `withSessionCookie(res, token)`/`clearSessionCookie(res)` (`Set-Cookie` header on the returned `Response`) — never `next/headers` `cookies()`, which is reserved for the `/dashboard` Server-Component guard. SIWE signatures are recovered and verified server-side with `siwe` + `viem` (EOA only in v1). Rate limiting is per-account lockout (`User.failedLoginCount` / `User.lockedUntil`) plus an in-memory sliding-window IP limiter (dev/single-instance only). The Auth UI is a client island that calls these routes and drives a terminal-style log pane with the real handshake steps.

**Tech Stack.** Next.js `15.1.0`, React `19.0.0`, TypeScript `5.6.3` (strict), Prisma `^6.19.3` + SQLite (dev) / Postgres (prod, via `DATABASE_URL`), Vitest `2.1.5` (jsdom default; Node per-file via `// @vitest-environment node`), Playwright `^1.61.1`. New runtime deps: `@node-rs/argon2` (Argon2id, prebuilt binaries), `siwe`, `viem`, `zod`. Package manager pnpm `10.33.0`, Node `>=20`.

## Global Constraints

Copy these verbatim into every reviewer's head; violating any one fails CI or the security posture.

- **Non-custodial, always (spec §1, §4, §5.7).** The server never stores, derives, transmits, or logs a private key, seed phrase, mnemonic, or plaintext password. There is no server-side signer. The only credential material persisted is an Argon2id password *hash* and short-lived SIWE nonces.
- **Secret-guard rule (`scripts/guard-no-secret-columns.sh`).** No Prisma column name may match `(privateKey|seedPhrase|mnemonic|plaintextPassword|passwordPlain|secretKey)` (case-insensitive). Password column MUST be named `passwordHash`. `pnpm guard:secrets` must stay green.
- **Password hashing.** `@node-rs/argon2`, Argon2id only. Never store plaintext. Params per spec §4.3: `memoryCost` ≥ 19 MiB (target 64 MiB → `65536` KiB), `timeCost = 3`, `parallelism = 1`, 16-byte salt, 32-byte output. Store the full encoded `$argon2id$...` string. `@node-rs/argon2` ships an install script → add it to `pnpm.onlyBuiltDependencies`.
- **Sessions.** Opaque, DB-backed. Token = `crypto.randomBytes(32)` hex; store only `sha256(token)` in `Session.tokenHash` (unique). Cookie `cr_session`: `httpOnly`, `secure` (prod), `sameSite=lax`, `path=/`, host-only, `maxAge` 30 days. Rotate on login (delete old row, issue new). Logout deletes the row. `revokeAllForUser(userId)` supported.
- **Cookie / request-context rule (CRITICAL — systemic).** Route handlers MUST NOT use `next/headers` `cookies()` (`.set()`/`.delete()`/`.get()`): it throws in the Vitest node env where tests invoke handlers with a bare `Request`. Instead: (a) READ the token with `getSessionFromRequest(req)` — parses `req.headers.get("cookie")` for `cr_session` and validates it (hash → DB lookup → expiry); (b) WRITE/CLEAR the cookie with `withSessionCookie(res, token)` / `clearSessionCookie(res)` from `lib/http/responses.ts`, which set a `Set-Cookie` header on the returned `Response` (Next honors `Set-Cookie` on returned Responses). The `next/headers` `cookies()`-based `getSession()` is reserved ONLY for Server Components (the `/dashboard` layout guard). Tests build a `Request` with a `cookie: "cr_session=<token>"` header and assert `res.headers.get("set-cookie")`.
- **SIWE.** `siwe` + `viem`, EOA signatures only (no EIP-1271 / RPC contract-wallet verification in v1). Nonce single-use via `SiweNonce` (short TTL ~5 min); bind `domain` + `uri` to our origin, validate `chainId` against allowlist, mark `consumedAt` atomically; reject replay / expired / wrong-domain.
- **CSRF.** `SameSite=Lax` cookies + an Origin/Referer allowlist check on EVERY state-changing (POST) route. Documented in `lib/auth/csrf.ts`.
- **Rate limiting / lockout.** Per-account: `failedLoginCount` + `lockedUntil` on `User` (lock after N=5 failures for 15 min). Per-IP: in-memory sliding-window limiter for auth endpoints. **NOTE loudly in code and plan: the in-memory limiter is dev/single-instance ONLY — replace with Redis/Upstash for multi-instance prod.**
- **Enumeration resistance (spec §4.3–4.4).** register/login return generic errors that never reveal whether an email exists. Login always runs an Argon2id verify (against a dummy hash if the user is absent) to equalize timing.
- **Validation.** `zod` on every route with `.strict()` (reject unknown fields). Normalize + lowercase email before lookup/store.
- **SQLite portability.** Prisma enums are declared as `String` columns with app-level TS union types + Zod validation (SQLite/Postgres portable; no native enum). State the documented value sets in `lib/auth/types.ts`. Monetary/integer chain values (none in this wave) would be decimal strings — not applicable here.
- **Conventions (match Wave 1 EXACTLY).** pnpm; TS strict, `@typescript-eslint/no-explicit-any` = error (NO `any`); unused vars prefixed `_` (`argsIgnorePattern`/`varsIgnorePattern` = `^_`); `import "server-only"` at the top of every server module (Vitest aliases `server-only`/`client-only` to `test/empty-module.ts`); server/db tests use `// @vitest-environment node` as the FIRST line; `DATABASE_URL` from env with `file:./dev.db` fallback; run `pnpm format` then `pnpm format:check` before every commit; new deps with install scripts go in `pnpm.onlyBuiltDependencies`; add ONE new Prisma migration and commit `prisma/migrations/`; the CI `web` workflow already runs `prisma migrate deploy`. Test files are named `*.test.ts(x)` under `lib/`, `test/`, or `components/`; e2e specs are `*.spec.ts` under `e2e/`.
- **CI gate (`.github/workflows/web.yml`).** After every task the full chain must pass: `guard:secrets` → `format:check` → `db:generate` → `prisma migrate deploy` → `typecheck` → `lint` → `test` → `build`. Keep the existing Home e2e (`e2e/home.spec.ts`) green.

  import type { User } from "@prisma/client";
  import { prisma } from "@/lib/db";

  export const MAX_FAILED = 5;
  export const LOCK_MS = 15 * 60 * 1000;

  export function isLocked(user: Pick<User, "lockedUntil">, now: Date = new Date()): boolean {
    return user.lockedUntil !== null && user.lockedUntil.getTime() > now.getTime();
  }

  export async function registerFailedLogin(userId: string): Promise<void> {
    const user = await prisma.user.update({
      where: { id: userId },
      data: { failedLoginCount: { increment: 1 } },
    });
    if (user.failedLoginCount >= MAX_FAILED) {
      await prisma.user.update({
        where: { id: userId },
        data: { lockedUntil: new Date(Date.now() + LOCK_MS) },
      });
    }
  }

  export async function resetFailedLogins(userId: string): Promise<void> {
    await prisma.user.update({
      where: { id: userId },
      data: { failedLoginCount: 0, lockedUntil: null },
    });
  }

  // DEV / SINGLE-INSTANCE ONLY. Process-local memory is not shared across instances;
  // replace with Redis/Upstash before multi-instance production. See spec §4.4, §4.9.
  const hits = new Map<string, number[]>();

  export function rateLimit(
    key: string,
    limit: number,
    windowMs: number,
  ): { ok: boolean; retryAfterSec: number } {
    const now = Date.now();
    const cutoff = now - windowMs;
    const recent = (hits.get(key) ?? []).filter((t) => t > cutoff);
    if (recent.length >= limit) {
      const retryAfterSec = Math.ceil((recent[0] + windowMs - now) / 1000);
      hits.set(key, recent);
      return { ok: false, retryAfterSec: Math.max(1, retryAfterSec) };
    }
    recent.push(now);
    hits.set(key, recent);
    return { ok: true, retryAfterSec: 0 };
  }

  export function __resetRateLimit(): void {
    hits.clear();

  import { cookies } from "next/headers";
  import type { Session, User } from "@prisma/client";
  import { SESSION_COOKIE, validateSessionToken } from "./session";
  import { unauthorized } from "@/lib/http/responses";

  // Server Components ONLY (next/headers throws outside a request scope, e.g. the Vitest node env).
  export async function getSession(): Promise<{ session: Session; user: User } | null> {
    const store = await cookies();
    const token = store.get(SESSION_COOKIE)?.value;
    if (!token) return null;
    return validateSessionToken(token);
  }

  // Route handlers: read + validate the cr_session token from the request's Cookie header.
  function readCookie(req: Request, name: string): string | undefined {
    const header = req.headers.get("cookie");
    if (!header) return undefined;
    for (const part of header.split(";")) {
      const [k, ...v] = part.trim().split("=");
      if (k === name) return decodeURIComponent(v.join("="));
    }
    return undefined;
  }

    req: Request,
  ): Promise<{ session: Session; user: User } | null> {
    const token = readCookie(req, SESSION_COOKIE);
    if (!token) return null;
    return validateSessionToken(token);
  }

  export async function requireSession(req: Request): Promise<{ session: Session; user: User }> {
    const s = await getSessionFromRequest(req);
    if (!s) throw unauthorized();
    return s;

  import { createSession } from "@/lib/auth/session";
  import { isLocked, registerFailedLogin, resetFailedLogins } from "@/lib/auth/lockout";

    if (!parsed.success) return genericAuthError();

    const email = normalizeEmail(parsed.data.email);

    // Always run a verify to equalize timing (enumeration resistance).
    const hash = user?.passwordHash ?? DUMMY_HASH;
    const passwordOk = await verifyPassword(hash, parsed.data.passphrase);

    if (!user || !user.passwordHash) return genericAuthError();

    // Lockout is a time window. If still active, reject. If it EXPIRED, reset the
    // failed count so the user starts fresh (see documented semantics above).
    if (isLocked(user)) return genericAuthError();
    if (user.lockedUntil !== null) {
      // lockedUntil set but no longer in the future → lock expired; clear the counter before re-checking.
      await resetFailedLogins(user.id);
    }

    if (!passwordOk) {
      await registerFailedLogin(user.id);
      return genericAuthError();
    }

    await resetFailedLogins(user.id);
    const { token } = await createSession(user.id, {
      userAgent: req.headers.get("user-agent") ?? undefined,
    });
    return withSessionCookie(json({ ok: true, next: "/dashboard" }), token);

  export const ALLOWED_CHAIN_IDS: readonly number[] = [DEFAULT_CHAIN_ID];

    return process.env.NEXT_PUBLIC_APP_URL ?? "http://localhost:3000";

    try {
      return new URL(appUrl()).host;
    } catch {

  export function appUri(): string {
    try {
      return new URL(appUrl()).origin;
    } catch {
      return "http://localhost:3000";
    }
  }

  export class SiweError extends Error {}

  export async function issueNonce(): Promise<string> {
    const nonce = generateSessionToken().slice(0, 32); // alphanumeric-ish hex, ≥8 chars per EIP-4361
    await prisma.siweNonce.create({
      data: { nonce, expiresAt: new Date(Date.now() + SIWE_NONCE_TTL_MS) },
    });
    return nonce;

  export interface SiweVerifyResult {
    user: User;
    address: string;
  }

  export async function verifySiwe(message: string, signature: string): Promise<SiweVerifyResult> {
    let siwe: SiweMessage;
    try {
      siwe = new SiweMessage(message);
    } catch {
      throw new SiweError("Malformed SIWE message.");
    }

    // Bind BOTH domain and uri to our origin (uri check is explicit — siwe.verify does not enforce it by default).
    if (siwe.domain !== appHost()) throw new SiweError("Domain mismatch.");
    if (siwe.uri !== appUri()) throw new SiweError("URI mismatch.");
    if (!ALLOWED_CHAIN_IDS.includes(siwe.chainId)) throw new SiweError("Chain not allowed.");

    // Cryptographic verification (EOA only; no EIP-1271/RPC lookup in v1).
    // siwe v2 `verify()` resolves to { success, data, error } and does NOT throw on a bad
    // signature — it sets success=false. Pass `time` so expirationTime/notBefore are enforced.
    // We also pass domain + nonce so the library cross-checks them against the message fields.
    let fields;
    try {
      fields = await siwe.verify({
        signature,
        domain: appHost(),
        nonce: siwe.nonce,
        time: new Date().toISOString(),
      });
    } catch {
      // Defensive: some inputs (malformed signature) can still throw.
      throw new SiweError("Signature verification failed.");
    }
    if (!fields.success) throw new SiweError("Signature verification failed.");

    // Single-use nonce: consume atomically. updateMany with usedAt=null guard prevents replay.
    const consumed = await prisma.siweNonce.updateMany({
      where: { nonce: siwe.nonce, usedAt: null, expiresAt: { gt: new Date() } },
      data: { usedAt: new Date(), address: getAddress(siwe.address) },
    });
    if (consumed.count !== 1) throw new SiweError("Nonce missing, used, or expired.");

    const address = getAddress(siwe.address);
    const existing = await prisma.linkedWallet.findUnique({
      where: { address },
      include: { user: true },
    });
    if (existing) {
      await prisma.linkedWallet.update({ where: { address }, data: { verifiedAt: new Date() } });
      return { user: existing.user, address };
    }

    const user = await prisma.user.create({
      data: {
        linkedWallets: { create: { address, chain: "EVM", verifiedAt: new Date() } },
        application: { create: { status: "DRAFT" } },
      },
    });
    return { user, address };
  }

  ```ts
  import { requireSession } from "@/lib/auth/guard";
  import { applicationSchema } from "@/lib/validation/auth";
  import { isAllowedOrigin } from "@/lib/auth/csrf";
  import { prisma } from "@/lib/db";
  import { json, badRequest, forbidden } from "@/lib/http/responses";

  export async function GET(req: Request): Promise<Response> {
    let session;
    try {
      ({ session } = await requireSession(req));
    } catch (res) {
      if (res instanceof Response) return res;
      throw res;
    }
    const application = await prisma.citizenshipApplication.findUnique({
      where: { userId: session.userId },
      select: { id: true, status: true, name: true, domicileCity: true, hostCountry: true, motto: true },
    });
    return json({ application });
  }

  export async function POST(req: Request): Promise<Response> {
    if (!isAllowedOrigin(req)) return forbidden();
    let userId: string;
    try {
      ({ user: { id: userId } } = await requireSession(req));
    } catch (res) {
      if (res instanceof Response) return res;
      throw res;
    }
    let body: unknown;
    try {
      body = await req.json();
    } catch {
      return badRequest();
    }
    const parsed = applicationSchema.safeParse(body);
    if (!parsed.success) return badRequest("Please check the application fields.");

    const application = await prisma.citizenshipApplication.upsert({
      where: { userId },
      update: { ...parsed.data },
      create: { userId, status: "DRAFT", ...parsed.data },
      select: { id: true, status: true, name: true, domicileCity: true, hostCountry: true, motto: true },
    });
    return json({ ok: true, application });
  }
  ```
  > NOTE the `catch (res)` pattern: `requireSession(req)` throws a `Response` (401). Because ESLint forbids `any`, the catch binding is `unknown`; narrow with `if (res instanceof Response) return res; throw res;` (already applied in both handlers above) — never a bare `as Response`.
